📖 概念速記

  • CLAUDE.md：放在專案資料夾根目錄，Claude Code 每次啟動對話都會先讀它。用來放專案背景、檔案結構、編碼規範與環境指令。
  • Plan Mode：規劃模式。此模式下 Claude 只討論、出計畫，不修改檔案。改計畫比改程式便宜十倍，切記在動手前先對齊計畫。
  • 檔案引用（@ 符號）：在對話框中打上 @ 即可選取並引用特定的檔案或資料夾。這能將你希望 AI 參考的特定文件或程式碼上下文直接餵給它，例如：「參考 @cleaner.py 的清理風格來撰寫 features.py」。
  • 小步迭代：一次只做一個小功能，執行確認無誤後，再做下一個。
  • 安全網：Desktop 的 diff 畫面點 Reject，或 CLI 的 /undo。可隨時撤銷 AI 上一次的檔案變更。
  • 上下文管理（防 AI 混亂）：當對話變長，AI 會開始變慢或產生幻覺。此時可打 /compact 壓縮並摘要歷史對話，或直接「停損重開」（Desktop 開啟 + New Session，CLI 輸入 /clear 或重啟），保持 AI 大腦清醒。

檔案 欄位

💡 三個一定要先知道的事
  • 金額 = 單價 × 數量，是個恆等式。而 單價 只是 10 種產品的固定價目表。這件事會直接決定今天的題目怎麼設計，往下看。
  • 兩張表靠 門市 join。兩張表都有 城市，join 時會撞名，要處理。
  • 沒有「是不是會員」這個欄位——只有 會員編號，空白代表非會員。想用它就得自己推導。

今天要預測什麼——以及為什麼不是「預測單筆訂單金額」

直覺上你會想：「有銷售資料，那就來預測每筆訂單的金額吧。」

這個題目不成立。因為 金額 = 單價 × 數量 是恆等式：

如果你餵給模型的特徵 R² 結果
包含 單價 和 數量 1.0000 完美預測，但這是資料洩漏——你等於把答案抄給它
不含 單價、數量 -0.06 比直接猜平均值還爛，因為資訊真的不夠
⚠️ 這是這門課想讓你帶走的判斷力

如果你叫 AI「幫我預測訂單金額」，它會很樂意照做，然後給你一個 R² = 1.0 的完美模型，還會恭喜你。它不會主動告訴你這是洩漏。

看到 R² 高得不像話，第一反應應該是懷疑，不是開心。

所以今天的題目是：預測「每間店每天的營收」

把交易明細聚合成「每間店每天一列」之後，題目就成立了：

項目 內容
預測目標 日營收＝這間店這天的所有交易金額加總
可用特徵 門市、店型、坪數、座位數、星期幾、月份、是否週末
資料筆數 6 間店 × 181 天 ≈ 1,086 筆
商業意義 「這間店下週六大概會做多少營收？」——排班、備料真的用得上
💡 這題也接得上你昨天做的坪效分析

昨天你發現坪數跟營收有關。今天你會親眼看到坪數就是特徵重要性的第一名——你昨天的直覺，今天被模型量化了。

🔬 機器學習管線架構（ML Pipeline）

在研究或業界，機器學習開發很少在單一 Jupyter Notebook 裡一寫到底。為了讓程式碼可重用、可測試，我們把工作流拆成模組化的管線。今天就照這個架構做：

各模組的職責

模組 做什麼
cleaner.py 讀取兩張表，去除重複列、補缺值、統一城市寫法與日期格式、標記退貨，並 join 門市資訊
features.py 把交易明細聚合成「每店每天一列」，加上時間特徵（星期幾、月份、是否週末）與門市屬性，並對類別欄位編碼
train.py 切分訓練／測試集，交叉驗證評估泛化能力，比較兩種模型，輸出效能圖表並保存模型
predict.py 載入訓練好的模型與編碼器，輸入門市與日期，即時預測當日營收

🔸 Round 2 — 建立 CLAUDE.md 家規

啟動後輸入以下指令建立專案規章：

請幫我建立 CLAUDE.md 專案規範檔案：

【專案目標】
用 Python、Pandas 與 scikit-learn 預測山嵐咖啡「每間門市每天的營收」。

【資料】
- sales_2026H1.csv 欄位：訂單編號、日期、門市、城市、產品、類別、單價、數量、金額、付款方式、會員編號
- stores.csv 欄位：門市、城市、開幕日期、坪數、店型、座位數
- 兩張表以「門市」join。注意兩張表都有「城市」欄位，join 時要處理欄位撞名。
- 皆為 utf-8-sig 編碼。

【重要限制】
- 金額 = 單價 × 數量 是恆等式，所以「單價」與「數量」絕對不可以當作預測日營收的特徵，會造成資料洩漏。
- 預測目標是聚合後的「日營收」，不是單筆訂單金額。

【檔案架構】
- cleaner.py：資料清理與門市資訊合併
- features.py：聚合成每店每天，並做特徵工程
- train.py：模型訓練、交叉驗證、指標評估與模型儲存
- predict.py：命令列互動預測腳本

【開發規範】
- 程式碼內部需加上詳細的繁體中文註解說明。
- 防禦性開發：一次僅寫完一個檔案或功能，向我確認無誤後，再前進下一步。

實作③：模型訓練、評估與互動預測

情境

資料整好了，現在回答那個真正有用的問題：這間店下週六大概會做多少營收？店長拿這個數字去排班、備料。

這題你會看到模型不是一次到位：先跑基準版本、看指標、再調整。你也會看到一件反直覺的事——比較複雜的模型不一定比較好。

先記住兩件事

💡 小步迭代與模組化

千萬不要一次叫 AI 寫完清洗、特徵、訓練、預測的所有程式——模型卡住時你會不知道是哪個環節出錯。一次只做一步，跑起來看指標對不對，再做下一個。

⚠️ 安全網：防禦性開發

訓練模型時，AI 經常會為了「優化」而引入過於複雜的套件或動到你的特徵。Desktop 每次修改都會先變成 diff 讓你看，看不順眼直接 Reject，還沒真的寫入檔案；CLI 打 /undo 撤銷。訓練出好的基準模型後就 git commit 存檔，後續改壞了隨時退回。

💡 @ 檔案引用：與其口頭描述，不如直接把檔案帶進來

接下來要叫 Claude 寫 train.py，它得先知道 cleaner.py／features.py 產出的資料長什麼樣（欄位名稱、型別）。與其用文字描述、靠它自己去猜或翻檔案（有時翻錯、有時漏看），不如在提示詞裡打 @，直接把檔案內容帶進這次對話——打 @ 會跳出路徑選單，選了就帶進去。下面的提示詞已經示範這個寫法。

🔸 Round 1 — 實作模型訓練與保存 train.py

請根據 @cleaner.py 與 @features.py 產出的每日營收資料欄位，實作 train.py：
1. 將資料集劃分為訓練集與測試集（test_size=0.2, random_state=42）。

cleaner.py——清理與合併

import pandas as pd

def clean_data(sales_path, stores_path):
    # 1. 讀取資料（utf-8-sig 才不會讓第一個欄位名帶 BOM）
    df_sales = pd.read_csv(sales_path, encoding='utf-8-sig')
    df_stores = pd.read_csv(stores_path, encoding='utf-8-sig')
    n0 = len(df_sales)

    # 2. 移除完全重複的列
    df_sales = df_sales.drop_duplicates()
    n_dup = n0 - len(df_sales)

    # 3. 補缺值：付款方式補「未註記」；數量或金額為空的直接丟棄
    n_na_pay = df_sales['付款方式'].isna().sum()
    df_sales['付款方式'] = df_sales['付款方式'].fillna('未註記')
    df_sales = df_sales.dropna(subset=['數量', '金額'])

    # 4. 統一城市寫法（台北／臺北、高雄／高雄市）
    city_map = {'臺北': '台北', '高雄市': '高雄'}
    df_sales['城市'] = df_sales['城市'].replace(city_map)
    df_stores['城市'] = df_stores['城市'].replace(city_map)

    # 5. 統一日期格式（資料裡混了 2026-05-24 與 2026/03/23 兩種寫法）
    df_sales['日期'] = pd.to_datetime(df_sales['日期'], format='mixed')

    # 6. 標記退貨（數量為負數）
    df_sales['是否退貨'] = df_sales['數量'] < 0

    # 7. 以「門市」join；兩張表都有「城市」，用 suffixes 避免撞名
    df = pd.merge(df_sales, df_stores, on='門市', how='left', suffixes=('', '_門市'))

    print(f"清理報告：移除重複列 {n_dup} 筆／付款方式補值 {n_na_pay} 筆／"
          f"標記退貨 {df['是否退貨'].sum()} 筆／join 後 {len(df)} 筆")
    return df

features.py——聚合與特徵工程

import pandas as pd
from sklearn.preprocessing import OneHotEncoder
import joblib

CATEGORICAL = ['門市', '店型']
NUMERIC = ['坪數', '座位數', '星期幾', '月份', '是否週末']

def build_daily(df):
    """把交易明細聚合成「每間店每天一列」。"""
    df = df[~df['是否退貨']]          # 訓練時不算退貨

    daily = (df.groupby(['門市', '日期'])
               .agg(日營收=('金額', 'sum'))
               .reset_index())

    # 門市屬性（每間店固定，取第一筆即可）
    attr = df.groupby('門市')[['坪數', '座位數', '店型']].first().reset_index()
    daily = daily.merge(attr, on='門市', how='left')

    return add_time_features(daily)

def add_time_features(daily):
    """從日期長出時間特徵。predict.py 也會用到，所以獨立成一個函式。"""
    daily = daily.copy()
    daily['日期'] = pd.to_datetime(daily['日期'])
    daily['星期幾'] = daily['日期'].dt.dayofweek      # 0=週一, 6=週日
    daily['月份'] = daily['日期'].dt.month
    daily['是否週末'] = (daily['星期幾'] >= 5).astype(int)
    return daily

class FeaturePipeline:
    def __init__(self):
        self.encoder = OneHotEncoder(handle_unknown='ignore', sparse_output=False)

    def fit(self, daily):
        # 只在訓練集上擬合編碼器，防止資料洩漏
        self.encoder.fit(daily[CATEGORICAL])

    def transform(self, daily):
        arr = self.encoder.transform(daily[CATEGORICAL])
        cols = self.encoder.get_feature_names_out(CATEGORICAL)
        encoded = pd.DataFrame(arr, columns=cols, index=daily.index)
        return pd.concat([encoded, daily[NUMERIC]], axis=1)

    def save(self, path):
        joblib.dump(self.encoder, path)

    def load(self, path):
        self.encoder = joblib.load(path)
⚠️ 注意特徵清單裡沒有「單價」「數量」「訂單數」

前兩個是恆等式的組成，放進去等於抄答案。訂單數 則是另一種洩漏——你預測明天營收時，並不知道明天會有幾筆訂單。

train.py——訓練、交叉驗證與評估

import numpy as np
import pandas as pd
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt
from sklearn.model_selection import train_test_split, KFold, cross_val_score
from sklearn.ensemble import RandomForestRegressor
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score, mean_absolute_error, mean_squared_error
import joblib
from cleaner import clean_data
from features import build_daily, FeaturePipeline

# 1. 讀取、清理、聚合成每日營收
df = clean_data('sales_2026H1.csv', 'stores.csv')
daily = build_daily(df)
print(f"聚合後：{len(daily)} 筆（每間店每天一列）")

# 2. 先切分，再擬合編碼器 — 順序反了就是資料洩漏
y = daily['日營收']
train_raw, test_raw, y_train, y_test = train_test_split(
    daily, y, test_size=0.2, random_state=42
)

# 3. 只用訓練集擬合特徵處理器
pipeline = FeaturePipeline()
pipeline.fit(train_raw)
X_train = pipeline.transform(train_raw)
X_test = pipeline.transform(test_raw)

# 4. K-Fold 交叉驗證
rf = RandomForestRegressor(n_estimators=100, max_depth=10, random_state=42)
kf = KFold(n_splits=5, shuffle=True, random_state=42)
cv = cross_val_score(rf, X_train, y_train, cv=kf, scoring='r2')
print(f"隨機森林 5-Fold CV R2: {np.mean(cv):.4f} (std: {np.std(cv):.4f})")

# 5. 訓練並與線性迴歸基準比較
rf.fit(X_train, y_train)
lr = LinearRegression().fit(X_train, y_train)

for name, model in [('隨機森林', rf), ('線性迴歸(基準)', lr)]:
    p = model.predict(X_test)
    print(f"{name:14s} R2={r2_score(y_test, p):.4f}  "
          f"MAE={mean_absolute_error(y_test, p):.0f} 元  "
          f"RMSE={np.sqrt(mean_squared_error(y_test, p)):.0f} 元")

# 6. 特徵重要性圖
imp = pd.Series(rf.feature_importances_, index=X_train.columns).sort_values()
plt.figure(figsize=(8, 5))
plt.barh(range(len(imp)), imp.values)
plt.yticks(range(len(imp)), imp.index)
plt.xlabel('Feature Importance')
plt.tight_layout()
plt.savefig('feature_importance.png', dpi=120)
plt.close()

# 7. 殘差圖
pred = rf.predict(X_test)
plt.figure(figsize=(6, 5))
plt.scatter(pred, y_test - pred, alpha=.4, s=14)
plt.axhline(0, color='r', lw=1)
plt.xlabel('Predicted')
plt.ylabel('Residual')
plt.tight_layout()
plt.savefig('residuals.png', dpi=120)
plt.close()

# 8. 保存
joblib.dump(rf, 'model.joblib')
pipeline.save('encoder.joblib')

請先在 Plan Mode 下為我設計網頁介面佈局與資料儲存的邏輯計畫。

沒靈感？選一題

  1. 論文引用次數預測器：用論文領域、作者數、關鍵字數與年份，預測未來被引用次數（迴歸）
  2. 學生成績預測器：根據學習時數、出席率、小考成績，預測學期總成績或是否及格（迴歸／分類）
  3. 顧客流失預測器：根據消費頻率、平均單價、最後消費時間，預測顧客是否會流失（分類）
  4. 二手筆電／手機估值器：根據品牌、使用年資、記憶體大小、外觀等級，預估二手售價（迴歸）
  5. 每日通勤時間預測器：根據出發時間、天氣、交通工具、星期幾，預測通勤分鐘數（迴歸）
  6. 會議論文錄取機率預估：輸入摘要字數、作者單位權重、研究主題，評估錄取機率（分類）
