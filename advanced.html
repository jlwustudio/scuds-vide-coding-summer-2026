這頁怎麼用

Day 2 你已經會了 Claude Code 的四支柱（CLAUDE.md、Plan Mode、小步迭代、安全網）。這頁把常用的進階技巧整理成六個獨立單元，彼此不相依，想看哪個看哪個。分成三群：

  • 立刻有感（①–④）：門檻低，桌面 App 就能用，今天做完核心軌就能試。
  • 打包重複流程（⑤ Skills）：把「每次都要交代的一串話」變成可重用的 / 指令。
  • 連真實資料（⑥ MCP）：從「上傳 CSV」升級到「直接連資料庫與雲端」。
💡 這頁的每個功能，你都可以直接問 Claude

工具一直在更新。最可靠的自學方式是問工具本身——在 Claude Code 裡打「這個功能現在怎麼用？」它內建了自己最新的文件。單元 ④ 就在講這件事。

① 貼截圖／圖表除錯

情境

你的 train.py 跑出一張殘差圖，但殘差好像有一條斜線的 pattern；或者 matplotlib 的中文標題變成一排「□□□」。這種「用講的講不清楚」的問題，直接把圖丟給 Claude 看最快。

是什麼：Claude Code 看得懂圖片——截圖、圖表、UI、手繪草圖、論文裡的表格都行。你不用把圖「描述」給它，直接給它看。

🖥️ Desktop App

把圖片直接拖曳進對話框，或複製後貼上。

⌨️ CLI

複製圖片後在終端機貼上：Windows/Linux 用 Ctrl+V、macOS（iTerm2）用 Cmd+V。或直接給路徑。

給路徑的講法（兩種介面都通）：

分析這張圖：/path/to/residuals.png

資料科學最實用的三種用法：

（附上 residuals.png）這是我模型的殘差圖，殘差有沒有系統性的 pattern？
如果有，通常代表我漏了什麼特徵？該怎麼改 features.py？
（附上一張跑歪的圖）這張 matplotlib 圖的中文變成方框、圖例還擋住資料點，
幫我修出圖的那段程式碼。
（附上論文裡的一張表格截圖）把這張表轉成一個 pandas DataFrame 的建立程式碼。
💡 什麼時候截圖、什麼時候複製文字

純文字的錯誤訊息，複製文字通常比截圖好（Claude 能精準讀到每個字）。截圖的主場是圖表、UI 畫面、示意圖、手繪草圖——這些用文字描述又慢又不準。

② @ 檔案引用

情境

你要 Claude 寫 features.py，希望它照你已經寫好的 cleaner.py 的風格與註解慣例。與其等它自己去翻檔案（有時翻錯、有時沒翻），不如一開口就把參考檔塞給它。

是什麼：在提示詞裡打 @，直接把某個檔案或資料夾的內容帶進這次對話。打 @ 會跳出路徑選單，選了就帶進去。

照 @cleaner.py 的風格與註解慣例，幫我寫 features.py。
根據 @data/資料集說明.md 的欄位規格，檢查 @train.py 有沒有用到不存在的欄位。
寫法 效果
@cleaner.py 把整個檔案內容帶進對話
@data/ 把該資料夾的檔案清單帶進來（只有列表，不是內容）
@server:resource 引用連好的 MCP 服務的資源（見單元 ⑥）
💡 順帶一提

用 @ 引用某個檔案時，Claude 會連同那個資料夾的 CLAUDE.md 一起帶進來——所以你的專案家規會跟著生效。

③ 上下文管理：/compact 與開新對話

情境

你為了調模型跟 Claude 來回聊了三十輪，它開始變慢、甚至忘記你一開始設好的編碼規範，或把前面某個舊錯誤一直帶著走。這不是它壞了——是對話太長，上下文塞太滿。

是什麼：這是 Day 1「鬼打牆就開新對話」的升級版。長對話會變慢、也更容易忘記前面的規則。兩個工具：

動作 什麼時候用 怎麼做
壓縮歷史 還在同一個任務，只是聊太長 → 保留重點、把囉嗦的過程摘要掉 輸入 /compact
開新對話 換一個新任務／已經鬼打牆 → 全部清空重來 Desktop 開新 session；CLI 輸入 /clear 或重啟
💡 開新對話不會弄丟你的家規

CLAUDE.md 每次啟動都會自動重讀，所以開新 session 後專案規矩還在。放心重開，這是保持 AI 清醒的衛生習慣，不是損失。

④ 讓 Claude 教你自己 ＋ /powerup

情境

你聽說 Claude Code 有很多功能，但不知道有哪些、也記不住指令。好消息：你不用記，直接問它。

是什麼：Claude Code 內建了自己最新版的說明文件，不管你裝的是哪個版本。不確定怎麼做，直接用白話問：

Claude Code 有哪些技能可以用？（what skills are available?）
要怎麼用 MCP 連我的資料庫？
Claude Code 可以幫我開 pull request 嗎？怎麼做？

另外兩個好用的內建指令：

  • /powerup — 內建的互動式教學，用動畫示範各種功能，最適合初次上手。
  • /help — 列出目前可用的指令。
💡 這才是面對「一直在變的工具」的正解

Claude Code 幾乎每週都在加功能。與其去背哪個部落格教的舊語法，問工具本身永遠是最新的。這頁教的每個技巧，你都可以再問它要更細的用法。

⑤ Skills：把重複的流程打包成 / 指令

情境

你發現自己每次都在對 Claude 交代同一串話：「用繁體中文、加詳細註解、先 Plan 再寫、報告一定要附 R²/MAE 表和特徵重要性圖」。與其每次重打，不如把它包成一個技能，以後打 /ml報告 就到位。

是什麼：Skill 就是一個放在資料夾裡的 SKILL.md，裝著一串你常用的指令或流程。你可以打 /技能名 直接叫它，Claude 也會在情境符合時自動用它。（官方已經把「自訂 slash 指令」併進 Skills，所以現代寫法就是 SKILL.md。）

💡 跟 CLAUDE.md 怎麼分工

「事實」放 CLAUDE.md（專案背景、欄位規格、用哪些套件）——每次啟動自動載入。「可重複的流程」包成 Skill（一套報告 SOP、一種清理步驟）——用到才載入，不佔平常的 context。當你發現 CLAUDE.md 裡某一段從「描述」長成了「步驟」，就該把它搬成 Skill。

怎麼做

建一個資料夾放 SKILL.md。放個人層（跨所有專案都能用）：

🖥️ Desktop / ⌨️ CLI 皆同

個人層路徑 ~/.claude/skills/<技能名>/SKILL.md；只想給這個專案用就放 .claude/skills/<技能名>/SKILL.md。資料夾名字就是你打的指令名。

其實最快的方式是叫 Claude 幫你建：

幫我建立一個名為 ml報告 的個人 skill（放在 ~/.claude/skills/ml報告/SKILL.md），
內容是：寫機器學習分析報告時，一律用繁體中文、程式碼加詳細註解、先用 Plan Mode 出計畫、
報告一定要附 R²/MAE/RMSE 表格與特徵重要性圖，並提醒我檢查資料洩漏。

它會產出類似這樣的 SKILL.md（開頭 --- 之間是 frontmatter，description 讓 Claude 知道何時該用）：

---
description: 產出機器學習分析報告。當我要求整理模型結果、寫分析報告，或做「對答案」時使用。
---

寫報告時一律遵守：
- 全程繁體中文，程式碼加上詳細註解
- 先用 Plan Mode 出計畫，我同意後再動手
- 報告一定要附：R² / MAE / RMSE 表格、特徵重要性圖、一段「模型限制」
- 交報告前提醒我檢查有沒有資料洩漏

之後在任何專案打 /ml報告 就會套用，或你一提到「幫我整理模型結果」它就自動用。

💡 你其實已經在用 Skills 了

Claude Code 內建的 /code-review、/debug 這些就是 skills。想更進階（測試、優化自己的技能）可以查官方的 skill-creator。

⑥ MCP：從「上傳 CSV」升級到「連真實資料」

情境

兩天課裡你都是把 CSV 上傳或拖進資料夾。但真實工作中，資料常常在資料庫或雲端裡。你不想每次都手動下 SQL、匯出 CSV、再上傳——你想直接叫 Claude「查這個月每家門市的營收」。這就是 MCP 在做的事。

是什麼：MCP（Model Context Protocol）是一個開放標準，讓 Claude Code 連上外部工具與資料源——PostgreSQL、Notion、Slack、GitHub 以及 Google Sheets (試算表) / Google Drive 等。連好之後，Claude 能直接讀它們、對它們動作（例如讀取或寫入資料），不用你當人肉搬運工。

🖥️ Desktop App

點 + → Connectors，選官方連接器（如 Google Drive / Sheets、Notion、GitHub），照畫面完成 OAuth 授權登入即可。最無痛。

⌨️ CLI 
# 遠端 HTTP 服務（例：Notion）
claude mcp add --transport http notion https://mcp.notion.com/mcp

# 新增 Google Sheets 連接器 (使用開源 MCP 伺服器)
claude mcp add google-sheets -- npx -y @smithery/mcp-server-google-sheets

# 管理與查狀態
claude mcp list
/mcp

連好之後就能這樣用：

查我們 PostgreSQL 上這個月每家門市的營收，聚合成每店每天，存成一份 CSV。
把我 Google Sheets 裡名為「專櫃業績通報」的試算表讀進來，做跟 Day 2 一樣的清理。
將今日預測的冷萃咖啡銷售結果，自動追加到 Google Sheets 中名為「每日預測記錄」表格的最後一列。
💡 Google Sheets MCP 運作原理

不論是 Desktop App 內建的 Google Drive 連接器，還是 CLI 使用的 Google Sheets MCP Server，背後都是透過 Google API 與 OAuth 授權。AI 會將你的自然語言意圖（例如「讀取 A1:B10 範圍」）轉化為對 API 的調用，實現雙向的資料讀取與自動回寫。

⚠️ 安全：這是把 AI 接上你的真實系統

MCP 給 Claude 存取你實際資料庫／雲端的權限。所以：連正式資料庫先用唯讀帳號或測試庫；每個動作看懂它要做什麼再放行；授權一律走官方或受信任的連接器，不要把敏感密碼直接貼進對話。

💡 不確定接什麼？問它

直接問「how do I use MCP with Claude Code?」或「幫我把本地的 SQLite 接成 MCP」，Claude 會帶你一步一步走（呼應單元 ④）。
