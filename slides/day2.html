先想清楚題目今天要預測什麼——為什麼不是「單筆訂單金額」

[陷阱] 直覺想預測「每筆訂單金額」

金額 = 單價 × 數量 是恆等式，單價 又只是固定價目表：

餵給模型的特徵R²
含 單價＋數量1.0000（洩漏＝抄答案）
不含 單價、數量−0.06（比猜平均還爛）

AI 會很樂意給你 R²=1.0 還恭喜你，不會主動說這是洩漏。R² 高得不像話，第一反應要是懷疑。

[題目] 改預測「每間店每天的營收」

把交易明細聚合成「每店每天一列」，題目就成立了：

預測目標日營收＝這間店這天所有交易金額加總
可用特徵門市、店型、坪數、座位數、星期幾、月份、是否週末
資料筆數6 店 × 181 天 ≈ 1,086 筆
商業意義「下週六大概做多少營收？」排班、備料用得上

[呼應] 昨天你發現坪數跟營收有關——等一下你會看到坪數就是特徵重要性第一名，直覺被模型量化了。

科學與工程並重機器學習專案模組化管線 (ML Pipeline)

1. 資料清理 [cleaner.py]

讀取原始 sales 與 stores 資料表，進行重複值去重、填補空值、統一文字拼寫，並 Join 合併門市維度資訊。

2. 特徵工程 [features.py]

先把交易明細聚合成「每店每天一列」算出日營收，再提取星期幾／月份／是否週末，對門市與店型做 One-Hot 編碼，並保存編碼器避免訓練-推理不一致。

3. 模型訓練 [train.py]

分割 Train/Test 數據，使用 5-Fold 交叉驗證評估隨機森林模型，計算評估指標，導出圖表並保存 model.joblib。

4. 預測推理 [predict.py]

載入已儲存的模型與編碼器，輸入門市與日期，即時預測這間店這天的營收。

[提示] 為什麼要模組化？ 研究生做研究常將程式寫在單一 Notebook 中，模組化管線是轉化為生產就緒 (Production-Ready) 工程代碼的第一步。

學術評估與決策基礎機器學習模型評估指標 (Regression Metrics)

指標名稱 數學定義 物理意義與解讀
決定係數 R² 1 - (Sum(y - y_pred)²) / Sum(y - y_mean)² 模型所能解釋的目標變數變異數比例。R² 越接近 1 擬合越好；為 0 等同猜平均值；小於 0 說明過擬合嚴重或模型結構錯誤。
平均絕對誤差 MAE Mean(|y - y_pred|) 預測值與實際值之間的平均誤差距離。單位與原數據一致（如元），不受平方放大，具備最佳商務可直覺解釋性。
均方根誤差 RMSE sqrt(Mean((y - y_pred)²)) 誤差的標準差。因為對誤差進行了平方，所以對大誤差/離群值極為敏感。若 RMSE 顯著高於 MAE，暗示有部分樣本預測嚴重偏離。

[分析] 殘差圖分析 (Residual Analysis)：殘差 e = y_actual - y_pred 的分佈應隨機且均勻地分佈在 y=0 兩側。若呈現明顯喇叭狀，代表模型在特定區間的誤差在放大，需要進一步特徵工程。

實作③ 實踐與存檔模型訓練與預測 — 5 輪步驟

🔸 R1–R3：模型訓練與指標出圖

  • R1 訓練與保存：實作 train.py 載入特徵、劃分數據並以隨機森林訓練，保存 model 與 encoder。
  • R2 交叉驗證：微調 train.py 加入 5-fold 驗證，並與基準 Linear Regression 比較兩者表現（結果可能出乎你意料）。
  • R3 指標與出圖：輸出 R²、MAE、RMSE，繪製並儲存特徵重要性與殘差圖 PNG。

🔸 R4–R5：推理腳本與 Git 存檔

  • R4 互動推理：實作 predict.py，載入儲存的模型，透過 CLI 輸入門市與日期即時預測當日營收。
  • R5 安全網與 Git：測試極端不合理 ML 要求，練習 Reject/undo，最後 git commit 存檔。

[警告] 環境卡住？ pip 安裝 scikit-learn 或執行 Python 卡住超過 3 分鐘，請果斷指示 Claude 轉為「單檔原生 Python 輕量迴歸管線（路線 B）」，無第三方套件依賴在終端機直接執行！

模型落地的第一步互動推理程式 predict.py 的運作原理

推理核心步驟

  1. 加載模型檔案：joblib.load('model.joblib') 與 joblib.load('encoder.joblib')
  2. 採集互動數據：透過 Python input() 詢問門市與日期（坪數、座位數、店型從 stores.csv 查表帶入，不用使用者輸入）。
  3. 特徵轉換對齊：將輸入數據轉為 1 列的 DataFrame，送入載入的特徵處理器，確保特徵欄位順序與編碼跟訓練集完全一致。
  4. 執行 Predict 預測：模型運算並輸出這間店這天的預測營收。

為何要獨立出推理檔案？

訓練 (train.py) 與預測 (predict.py) 是兩個生命週期不同的階段。

在實際生產環境中，模型往往是在伺服器上訓練好，然後打包發佈。預測端（可能是網頁、手機 APP 或 CLI 工具）只需要載入靜態的模型權重，並不需要重新讀取歷史 CSV 數據或重跑一次模型訓練流程。

進階觀念MCP 外部連接器是什麼？

Model Context Protocol (MCP)：就像為 AI 接上 「USB 擴充插槽」。

讓 AI 真正連接現實世界

  • 預設限制：AI 只能讀取本機上的文字與檔案，無法與外部互動。
  • MCP 賦能：透過統一的協定插槽，AI 可以直接連接 SQL 資料庫、存取 Google Drive 雲端檔案、搜尋網頁，甚至發送 Slack 通知與 Gmail 郵件！

如何使用它？

  • Claude Code 內建 MCP 客戶端。🖥️ Desktop：點 + 選 Connectors，圖形化設定。⌨️ CLI：終端機輸入 claude mcp add ...
  • 例如，若要讓 Claude Code 能隨時上網查最新套件文件，可以連接 Brave Search / Google Search 這類 MCP 服務。
